# Contributing Guide

Thanks for helping improve the Encrypted Markdown Reader!

## Getting Started
1. Fork the repo and create a feature branch.
2. Run `npm test` to validate encryption/decryption logic.

## Development Workflow
- Keep changes focused and documented.
- Update `README.md` when behavior changes.

## Reporting Issues
Please include:
- Steps to reproduce
- Expected vs actual behavior
- Browser and OS details

## Code Style
- Use readable variable names and avoid hidden side effects.
- Keep security-related logic isolated and well-commented.
